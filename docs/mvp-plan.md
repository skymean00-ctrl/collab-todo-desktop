## 1. MVP 목표

**카톡은 잡담/소통용으로 두고, "업무 지시 → 확인 → 처리 → 보고"의 흐름만 별도로 관리한다.**

### MVP가 해결하는 3가지 문제

| # | 문제 | MVP 해결 방식 |
|---|------|--------------|
| 1 | "말했다/안봤다" 분쟁 | 업무 수신 시 **확인 + 기한 약속** 필수 |
| 2 | 파일이 카톡에 묻힘 | 업무별로 파일이 **묶여서** 관리됨 |
| 3 | 진행 상태 불투명 | **대시보드**에서 전체 업무 현황 한눈에 |

---

## 2. 사용자 역할

| 역할 | 설명 | 권한 |
|------|------|------|
| **관리자** | 현장소장, 팀장 등 | 가입 승인, 업무 생성, 모든 업무 조회 |
| **팀원** | 사무직 담당자 | 업무 생성, 본인 관련 업무 조회/처리 |

---

## 3. 핵심 기능 (MVP 범위)

### 3.1 가입 / 로그인

```
[가입 신청] → [관리자 승인] → [로그인 가능]
```

- ID / 비밀번호 방식 (가장 단순)
- 가입 시: 이름, ID, 비밀번호 입력
- **관리자가 승인해야** 로그인 가능
- 승인 전에는 "승인 대기 중" 안내

### 3.2 업무 생성

업무를 만들 때 입력하는 항목:

| 항목 | 필수 | 설명 |
|------|------|------|
| 제목 | O | "3공구 견적서 작성" |
| 설명 | X | 상세 내용 (자유 입력) |
| 담당자 | O | 주 담당자 1명 지정 |
| 마감일 | O | 언제까지 완료해야 하는지 |
| 파일 첨부 | X | 관련 자료 첨부 |

> **설계 원칙**: 입력 항목을 최소화한다. 복잡하면 아무도 안 쓴다.

### 3.3 업무 흐름 (상태 관리)

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  새 업무  │ →  │   확인   │ →  │  진행중   │ →  │   완료   │
│ (등록됨) │    │ (수락함) │    │ (처리중) │    │ (보고함) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**각 단계 설명:**

1. **새 업무**: 업무가 등록되고 담당자에게 알림
2. **확인**: 담당자가 "확인" 버튼 + "○월 ○일까지 하겠습니다" 기한 약속
3. **진행중**: 실제 처리 중 (자료 요청, 취합 등)
4. **완료**: 담당자가 결과물과 함께 완료 보고

### 3.4 자료 요청 (하위 업무)

하나의 업무를 처리하기 위해 다른 사람에게 자료를 요청하는 기능:

```
[견적서 작성] (주 담당: 김과장)
  ├── [자료요청] 도면 보내주세요 → 이대리  ✅ 완료
  ├── [자료요청] 소요시간 확인 → 박주임  🔄 진행중
  └── [자료요청] 자재 단가표   → 최사원  ⏳ 확인 대기
```

- 주 담당자가 다른 팀원에게 **자료 요청** 생성
- 각 자료 요청도 동일한 흐름: 새 업무 → 확인 → 진행중 → 완료
- 자료 요청 완료 시 **파일 첨부** 가능 → 주 업무에 자동 취합

### 3.5 파일 관리

- 업무별로 파일이 **묶여서** 보임
- 업무 상세 화면에서 관련 파일 **한 곳에서** 확인
- 자료 요청에서 올린 파일도 주 업무에서 볼 수 있음
- 지원 형식: 문서(PDF, Excel, Word, HWP), 이미지(JPG, PNG), CAD 파일

### 3.6 대시보드 (메인 화면)

로그인 후 첫 화면에서 보이는 것:

```
┌─────────────────────────────────────────────┐
│  내 업무                          [새 업무+] │
│─────────────────────────────────────────────│
│                                             │
│  🔴 확인 필요 (2건)                          │
│  ┌─────────────────────────────────────┐    │
│  │ 3공구 견적서 작성    마감: 2/25      │    │
│  │ 자재 발주 확인       마감: 2/21      │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  🟡 진행중 (3건)                             │
│  ┌─────────────────────────────────────┐    │
│  │ 2공구 도면 검토      마감: 2/23      │    │
│  │ 월간 보고서 작성     마감: 2/28      │    │
│  │ 안전 점검표 취합     마감: 2/22      │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  🟢 완료 (5건)                    [더보기 >] │
│                                             │
│─────────────────────────────────────────────│
│  [내 업무]  [전체 현황]  [알림]              │
└─────────────────────────────────────────────┘
```

**전체 현황 탭** (관리자/팀장 주로 사용):
- 모든 업무의 상태를 한눈에
- 누가 어떤 업무를 맡고 있는지
- 마감 임박 / 지연 업무 강조 표시

### 3.7 알림

| 알림 종류 | 시점 |
|-----------|------|
| 새 업무 배정 | 나에게 업무가 배정됐을 때 |
| 자료 요청 | 누군가 나에게 자료를 요청했을 때 |
| 확인 독촉 | 배정 후 일정 시간 내 미확인 시 |
| 마감 임박 | 마감일 하루 전 |
| 완료 보고 | 내가 지시한 업무가 완료됐을 때 |

**알림 전달 방식 (단계적 구현):**

| 단계 | 방식 | MVP 포함 |
|------|------|----------|
| 1단계 | 앱 내 알림 (프로그램 내부) | ✅ |
| 2단계 | PC 데스크톱 팝업 알림 | ✅ |
| 3단계 | 카카오톡 알림 연동 | ⏳ (이후 버전) |

> 카카오톡 알림 연동은 카카오 비즈니스 API 연동이 필요하여 MVP 이후 추가 예정

## 6. 사용 시나리오 예시

### 시나리오: "3공구 견적서 만들어"

```
1. 김팀장이 [업무 생성]
   → 제목: "3공구 견적서 작성"
   → 담당자: 박과장
   → 마감: 2월 25일

2. 박과장에게 알림 도착 (PC 팝업)
   → 업무 열어서 [확인] + "2월 24일까지 하겠습니다"
   → 김팀장에게 "박과장이 확인했습니다" 알림

3. 박과장이 자료 요청 생성:
   → 이대리에게: "도면 보내주세요" (마감: 2/21)
   → 최사원에게: "자재 단가표 보내주세요" (마감: 2/22)

4. 이대리: 도면 파일 첨부 + [완료]
   최사원: 단가표 파일 첨부 + [완료]
   → 박과장에게 각각 "자료 도착" 알림

5. 박과장: 취합 후 견적서 작성, 파일 첨부 + [완료]
   → 김팀장에게 "견적서 작성 완료" 알림

6. 김팀장: 업무 상세에서 견적서 + 관련 자료 모두 확인 가능
```

**결과**: 누가 언제 확인했고, 어디서 멈춰있고, 파일은 어디있는지 전부 추적 가능.
