# 사용자 인터뷰 요약

> 인터뷰 일시: 2026-02-19

---

## 1. 배경 및 문제 정의

### 현재 상황
- 건설 현장 사무직 인원들이 **카카오톡**으로 업무 소통
- 여러 카톡방에 업무 지시, 보고, 질문, 파일 공유가 **전부 섞여** 있음

### 핵심 문제
- **메시지/파일이 묻힘**: 파일을 올려놨다고 하지만 그 순간 못 보면 찾는 데 더 오래 걸림
- **책임 분쟁**: "나는 말했다" / "안 본 니가 잘못" → 스트레스
- **흐름 불투명**: 지금 업무가 어디까지 진행됐는지 아무도 모름

## 2. 사용자 프로필

| 항목 | 내용 |
|------|------|
| 대상 | 건설 현장 사무직 |
| 인원 | 5~20명 |
| IT 숙련도 | 낮음 (IT에 익숙하지 않음) |
| 연령대 | 높은 편 |
| 사용 기기 | PC + 모바일 둘 다 |

---

## 3. 업무 흐름

### 기본 흐름
```
지시 → 담당자 처리 → 보고
```

### 실제 흐름 (더 복잡)
```
지시 (예: 견적서 작성)
  ├── A에게 도면 요청
  ├── B에게 현장 소요시간 요청
  ├── C에게 단가 요청
  └── 취합 → 견적서 작성 → 보고
```

- 하나의 업무에 관련 인원 수는 **그때그때 다름** (2~5명+)
- 시작점도 다양하지만 기본은 **지시 → 자료 수집/취합 → 결과물 → 보고**

---

## 4. 해결해야 할 3대 문제

### (1) 확인 추적
- 지시를 **봤는지** 확인
- **언제까지 하겠다**는 응답까지 받고 싶음
- "확인 + 기한 약속" 방식 (단순 읽음 확인이 아님)

### (2) 자료 정리
- 업무별로 파일이 **묶여야** 함
- 카톡처럼 채팅 흐름에 파일이 섞이면 찾을 수 없음

### (3) 진행 상태 파악
- 지금 **누가 뭘 하고 있는지** 한눈에 보여야 함
- 나도 알고 너도 알고 — 업무 흐름의 **투명성**

---

## 5. 설계 원칙 (제약 조건)

1. **카톡보다 복잡하면 안 됨** — 복잡하면 채택 실패
2. **연령대 높은 분도 설명 없이 사용 가능해야 함** — 직관적 UI
3. **전원 채택이 전제** — "나만 쓴다고 되는 문제가 아님"
4. **기존 도구(지라, 노션, 트렐로) 실패 경험** — 너무 복잡해서 포기
